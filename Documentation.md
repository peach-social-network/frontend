
## Git
#### Название ветки формируется:
`TASK-ID`

-   TASK-ID - Id задачи в Trello, без каких либо вложенностей (такие как: `feat/CP-1111` не приветствуются, просто `CP-1111`)

#### Коммит формируется:

`TASK-ID | description`

-   TASK-ID - Id задачи в Trello
-   description - краткое описание изменений

#### Merge requests 
Merge request формируется:

`Draft: TYPE: TASK-ID | description`

-   Draft - Указывает, что МР не должен временно мёржиться с целевой веткой. В случае ненадобности - удалить
    
    `TYPE: TASK-ID | description`
    
-   TYPE:
    
    -   **feat**：Новые фичи
    -   **fix**：bugfix
    -   **docs**：Обновление документации
    -   **style**：Модификации кода, не влияющие на логику программы (изменение пробельных символов, форматирование отступов, заполнение отсутствующих точек с запятой и т. д. без изменения логики кода)
    -   **refactor**：Рефакторинг кода (ни новых функций, ни исправлений ошибок)
    -   **perf**：Производительность, оптимизация
    -   **test**：Добавление новых тестов или обновление существующих
    -   **build**：Основная цель - изменить фиксацию системы сборки проекта (например, глюк, новый веб-пакет, конфигурация накопительного пакета и т.д.).
    -   **ci**：Основная цель - изменить коммиты, в которых проект продолжает процесс интеграции (например, Travis, Jenkins, GitLab CI, Circle и т. д.).
    -   **chore**：Другие типы, не принадлежащие к указанным выше типам
    -   **revert**：Откатить предыдущий коммит
-   TASK-ID - Id задачи в Trello
    
-   description - полное название задачи в Trello (заголовок)

## Architecture
Далее будет описано, по какому принципу и где какие файлы должны располагаться

#### Pages:
- Содержит в себе папки, которые называются с маленькой буквы, согласно назанию страницы - `users`, или `posts`. 
- Если для отображения страницы необходим какой-либо динамический параметр - в папке **pages** создается папка с названием страницы в единственном числе, и в ней создается файл [< param name >].tsx. 
#### Page:
- Содержит в себе три файла: **index.tsx**, **< pagename >.test.tsx**, **style.ts or style.module.sass**
- Должна называться с маленькой буквы, если это страница, содержащая какой то список, тогда в множественном числе, например `users`
#### Components
- Все компоненты в этой папке делятся на две группы - `common` - необходимые в нескольких местах приложения, и `< page name >`, т. е компоненты, которые относятся только к определенной страницы
- В каждой из этих папок находится две папки - `ui` и `functional`. Компонент определяется в `ui`, если он не содержит в себе никакой логики запросов и не затрагивает store приложения, т.е служит только для отображения данных. Компонент должен находиться в `functional`, если запрашивает, или отправляет какие-либо данные, или взаимодействует с хранилищем приложения
#### Component
- Содержит в себе Содержит в себе три файла: **index.tsx**, **< componentName >.test.tsx**, **style.ts or style.module.sass**
- Должен находиться, как описано  [тут](####Components) (В разделе Components)
#### Store
- У приложения может быть несколько хранилищ, при необходимости использовать хранилище надо обернуть компонент, в котором используется хранилище, в observer() и импортировать в этот компонент нужный store. Пример создания хранилища можно посмотреть по пути `store/store.ts`
- Store не должен публично предоставлять доступ к полям внутри него, для получения, или изменения поля необходимо создать функции, позволяющие получить необходимое поле
#### services
- Содержит файлы, которые делают какие либо запросы
- Файл должен называться по принципу: < Модель, которую он запрашивает>Service. Пример `UserService`. Должен называться именно в единственном числе
#### service
- Файл должен содержать создание нового экземпляра класса Api, который лежит по пути `services/config.ts`
- Должен по умолчанию экспортировать класс, содержащий несколько статических методов
- Метод называется по принципу - < request method > + то, что он получает, например: 
	 ```javascript 
		const api = new Api()
		class UserService {
			getAll(){
				api.request(
					{
						method: "get",
						url: "api/users"
					}
				)
			}
			getOne(id: number){
				api.request(
					{
						method: "get",
						url: `api/user/${id}`
					}
				)
			}
		}
		export default UserService
	```
#### types
- Содержит типы, которые необходимы в нескольких местах приложения
- Файлы должны называться в единственном числе в виде < имя >.type.ts: `user.type.ts`
- Каждый файл содержит интерфейс, или тип, который называется так же, как имя файла, если это интерфейс, то начиная с I, если type, то просто имя: **Интерфейс**: `interface IUser {}`. **Тип**: `type User = {}`  

## Code Style
- Весь код должен писаться в стиле `camelCase`,     
- Методы-обработчики именуются:  `{{цель}}{{действие}}Handler`. Пример:  `userChangeHandler`.
- Методы-предикаты именуются:  `is{{цель}}{{условие}}`. Пример:  `isModalVisible`.
- Методы-действия именуются:  `{{глагол}}{{контекст}}`. Пример:  `getUser`,  `generateStyle`  и т.д. Так же глагол должен использоваться один и тот же во всех методах-действиях одного типа. Например, получение чего-либо - всегда  `get...`, не использовать синонимы, по типу  `take`  и прочие. Если  `generate`  - всегда  `generate`, не нужно мешать с, например,  `make`.
- Компоненты и страницы должны содержать стрелочную функцию и экспортировать ее по умолчанию. Необходимо указать, что это функциональный, или классовый компонент, поэтому после объявления функции надо указать ее тип.  Пример:  
	```javascript 
	const SomeComponent:FC = () => {
	return <div></div>
	}
	export default SomeComponent
	```
- Если у компонента есть пропсы - необходимо типизировать их интерфейсом. Он должен объявляться перед объявлением функции, называться IProps и указываться В виде functionName:FC< IProps >:
	```javascript 
	interface IProps {
		someProps: string
	}
	const SomeComponent:FC<IProps> = ({someProps}) => {
	return <div>{someProps}</div>
	}
	export default SomeComponent
	```
- Стили пишутся с помощью styled-componens, все цвета, шрифты и размеры должны импортироваться из файла с соответсвующим названием:
	```javascript 
	//constants/colors.ts
	
	export const colors = {
		gray: {
			light: "lightgray"
		}
	}
	```
	```javascript 
	//<component>/style.ts
	
	export const Title = `
		color: ${colors.gray.light};
	`
	```
